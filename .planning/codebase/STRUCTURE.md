## Directory Purposes

**`renderer/`:**
- Purpose: The widget's entire front-end — markup, logic, styles, and translations
- Contains: Four JS/HTML/CSS files loaded as plain scripts (no bundler)
- Key files: `app.js` (all UI behaviour), `locales.js` (must load before `app.js`)

**`assets/`:**
- Purpose: Runtime icons bundled into the installer via `electron-builder` `files` config
- Contains: ICO (Windows), PNG and SVG variants for app icon and tray icon
- Key files: `icon.ico` (Windows installer), `tray.png` (system tray)

**`build/`:**
- Purpose: Build-time assets consumed by `electron-builder` for macOS and Linux packaging
- Contains: `icon.png` at 1024×1024 — generated by CI if absent, or by `npm run icons`
- Key files: `icon.png`

**`scripts/`:**
- Purpose: Developer utility scripts run manually, not shipped in the app
- Contains: `gen-icons.js` — uses `@resvg/resvg-js` and `png-to-ico` to regenerate icons from SVGs
- Key files: `gen-icons.js`

**`.github/workflows/`:**
- Purpose: CI/CD pipeline
- Contains: `build.yml` — parallel builds for Windows (NSIS), macOS (DMG+ZIP), Linux (AppImage); tag-triggered GitHub Release publish

**`.planning/codebase/`:**
- Purpose: GSD codebase map documents consumed by `/gsd-plan-phase` and `/gsd-execute-phase`
- Generated: Yes (by `/gsd-map-codebase`)
- Committed: Yes

## Key File Locations

**Entry Points:**
- `main.js`: Electron main process entry; `app.whenReady()` at line 515 bootstraps all windows
- `renderer/index.html`: Widget HTML; loads `locales.js` then `app.js` via `<script>` tags
- `renderer/app.js`: Renderer entry; `init()` called at line 506 on script load

**Configuration:**
- `package.json`: `"main": "main.js"`, `electron-builder` config (appId, targets, publish), `scripts`
- `.github/workflows/build.yml`: CI matrix configuration

**Core Logic:**
- `main.js`: All window creation, IPC handlers, data extraction loop, alert system, cookie management, auto-updater
- `preload.js`: Complete `window.claudeAPI` surface definition
- `renderer/app.js`: All UI state, panel management, usage rendering, settings serialization
- `renderer/locales.js`: All user-visible strings; add new locales here

**Icons/Assets:**
- `assets/tray.png`: System tray icon (must be non-empty; fallback inline base64 in `main.js` L391)
- `assets/icon.ico`: Windows-only app icon
- `build/icon.png`: macOS and Linux app icon (1024×1024)

## Naming Conventions

**Files:**
- `camelCase.js` for scripts: `main.js`, `preload.js`, `app.js`, `locales.js`, `gen-icons.js`
- `kebab-case.html/css` for web assets: `index.html`, `style.css`
- Locale suffix for translated docs: `README.pt-BR.md`

**Directories:**
- Lowercase single-word: `renderer/`, `assets/`, `build/`, `scripts/`, `docs/`

**CSS:**
- IDs with hyphens: `#status-dot`, `#usage-list`, `#settings-panel`
- Classes with hyphens: `.usage-item`, `.bar-fill`, `.theme-opt`, `.icon-btn`
- CSS custom properties prefixed with `--`: `--bg`, `--text-primary`, `--accent-blue`

**JavaScript variables:**
- `camelCase` for all variables and functions: `mainWindow`, `hiddenWindow`, `scheduleFetch`, `extractOnce`
- `UPPER_SNAKE_CASE` for module-level constants: `EXTRACT_SCRIPT`, `RENDER_DELAY_MS`, `CLAUDE_ORIGIN`, `USAGE_URL`
- `ALL_CONTENT_PANELS` for constant arrays

**IPC channel names:**
- `kebab-case` strings: `'usage-data'`, `'auth-status'`, `'next-fetch-at'`, `'open-login'`, `'get-settings'`, `'save-settings'`

**i18n keys:**
- `camelCase` in `LOCALES` object: `'authMsg'`, `'loginBtn'`, `'intervalHint'`

## Where to Add New Code

**New IPC command (main ↔ renderer):**
1. Add handler in `main.js`: `ipcMain.handle('command-name', ...)` or `ipcMain.on('command-name', ...)`
2. Expose in `preload.js`: add method to `contextBridge.exposeInMainWorld('claudeAPI', {...})`
3. Call from renderer via `window.claudeAPI.methodName()`

**New UI panel:**
1. Add HTML markup in `renderer/index.html` inside `<div id="body">`
2. Add panel to `ALL_CONTENT_PANELS` array in `renderer/app.js` if it should be hidden when other panels show
3. Add CSS in `renderer/style.css`

**New settings field:**
1. Add HTML input in the `#settings-panel` section of `renderer/index.html`
2. Read/write in `openSettings()` and `saveSettings()` in `renderer/app.js`
3. Consume in `main.js` via the `save-settings` IPC handler (already calls `store.set('settings', settings)`)
4. Add i18n keys for labels in both locales in `renderer/locales.js`

**New alert type:**
1. Add UI controls in `#settings-panel` in `renderer/index.html`
2. Read/write in `openSettings()` / `saveSettings()` in `renderer/app.js`
3. Add evaluation logic in `checkAlerts()` in `main.js`
4. Add state tracking key to `alertState` object (`main.js` L19)

**New locale:**
1. Add new language object to `LOCALES` in `renderer/locales.js` (copy `'en'` structure)
2. Add a `<button class="theme-opt" data-lang="xx">` in `renderer/index.html` settings panel

**New icon variant:**
1. Edit SVG sources in `assets/`
2. Run `npm run icons` to regenerate PNG and ICO files via `scripts/gen-icons.js`
